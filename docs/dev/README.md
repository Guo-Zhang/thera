# 架构设计文档

## 领域模型

### 社区（Community）

基本定义见知识工程标准。这里根据业务需求对社区类别进行拓展。

社区类别定义：

| 社区ID | 名称 | 描述 | 主要节点类型 |
|--------|------|------|-------------|
| Community 1 | 自我探索 | 记录个人情感、成长经历、内心独白等私密内容 | 个人情感、成长经历、反思日志 |
| Community 2 | 工作思考 | 记录职业发展、项目经验、技能学习等工作相关内容 | 项目经验、技能树、职业规划 |
| Community 3 | 小说创作 | 构建虚构世界，包含角色、情节、世界观等创作元素 | 虚构角色、故事线、世界观设定 |

## 上下文

### 小说创作

在小说创作场景中，三个社区协同工作：

1. 自我对话记录在Community 1（自我探索）
2. 工作思考记录在Community 2（工作思考）
3. Community 3（小说创作）通过人物原型映射链接前两个社区的内容：
   - 虚构角色 ↔ 真实个体映射
   - 情节灵感 ↔ 个人经历映射
   - 世界观构建 ↔ 现实观察映射

# 元模块

## 分类器

### 分类法：ADAS模型

1. Activity（活动）—— 取代 Project
定义：当前的执行流与焦点。
本质：动态的“做”。
特征：短周期、高交互。它不再是冷冰冰的项目档案，而是 AI 外脑当前的“工作记忆区”。用户打开工具即进入 Activity。
AI 逻辑：决定 System 1（快思考）的上下文窗口和即时响应策略。

2. Domain（领域）—— 取代 Area
定义：知识边界与责任范畴。
本质：静态的“语境”。
特征：长周期、稳定性。它划定了知识检索的命名空间（如“后端开发”、“产品设计”）。
AI 逻辑：决定 System 2（慢思考）的 RAG 检索范围和专业知识库的加载。

3. Asset（资产）—— 整合 Resource
定义：所有有价值的信息实体。
本质：被操作的“内容”。
特征：统一管理。不再区分“原材料”还是“产出物”，只要有价值，皆为资产。包括文档、代码、笔记、网页剪藏等。
AI 逻辑：向量化的数据源，AI 的长期记忆库。

4. State（状态）—— 重构 Archive
定义：附加在实体上的元数据属性。
本质：流动的“阀门”。
特征：维度升级。您将“归档”从一个“地点”降维成了一个“状态”。
生命周期状态：Active（活跃） / Archived（已归档）。
可见性状态：Public（公开） / Private（私有）。
操作权限：Read-Only（只读） / Read-Write（读写）。
AI 逻辑：数据过滤的护城河。确保 AI 只在授权范围内检索，自动屏蔽归档数据的干扰。

### 默认标签

活动：工作、生活、自我、创造等。
领域：思考、写作、编程等。
资源：飞书知识库、GitHub仓库等。
状态：是否归档、是否公开、是否可读等。

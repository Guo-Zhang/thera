# vital 交互设计框架

> 版本：0.0.1 | 更新日期：2026-03-21

## 1. 设计定位

### 1.1 观察者视角

vital 是 thera 的 GUI 前端，定位于**观察者**角色：

| 角色 | 工具 | 交互方式 |
|------|------|----------|
| 操作者 | thera (CLI) | 执行命令、写入数据 |
| 观察者 | vital (GUI) | 浏览数据、理解状态 |

**核心原则：**
- 展示 > 操作
- 状态 > 细节
- 时间线 > 快照

### 1.2 与 thera 的关系

```
┌─────────────────────────────────────────────┐
│                meta/profile/                │
│   submodules.yaml │ release.md │ memory.md  │
└─────────────────────────────────────────────┘
         ▲                      ▲
         │ 读取                 │ 读取
         │                      │
┌────────┴────────┐    ┌───────┴───────┐
│      thera      │    │     vital     │
│   CLI 操作工具   │    │   GUI 浏览器  │
└─────────────────┘    └───────────────┘
```

- 共享同一数据源
- 独立运行，互不依赖
- thera 写入，vital 读取

### 1.3 目标用户

| 用户 | 场景 | 需求 |
|------|------|------|
| 创始人 | 日常回顾 | 快速了解项目全貌 |
| 团队成员 | 新人入门 | 理解项目结构和规范 |
| AI Agent | 辅助决策 | 获取结构化数据 |

## 2. 信息架构

### 2.1 数据源映射

| 数据源 | 格式 | GUI 页面 | 优先级 |
|--------|------|----------|--------|
| `meta/profile/submodules.yaml` | YAML | 子模块总览 | P0 |
| `meta/profile/release.md` | Markdown 表格 | Release 时间线 | P1 |
| `meta/profile/memory.md` | Markdown | 记忆框架可视化 | P2 |
| `meta/journal/*.md` | Markdown | 工作日志时间线 | P2 |

### 2.2 页面层级

```
vital
├── 主页 (Home)
│   ├── 子模块总览 (Submodules)      ← 已实现
│   ├── Release 时间线 (Releases)    ← 规划中
│   ├── 记忆框架 (Memory Grid)       ← 规划中
│   └── 工作日志 (Journal)           ← 规划中
└── 关于 (About)
```

### 2.3 导航体系

```
┌─────────────────────────────────────────┐
│  ← 返回        vital        [菜单]     │  ← 顶部导航栏
├─────────────────────────────────────────┤
│                                         │
│              页面内容                   │
│                                         │
└─────────────────────────────────────────┘
```

**导航规则：**
- 主页提供快捷入口按钮
- 子页面左上角返回按钮
- 无深层嵌套，最多两层

## 4. 组件规范

### 4.1 字体

| 类型 | 字体 | 字号 |
|------|------|------|
| 标题 | Noto Sans CJK SC | 24dp |
| 正文 | Noto Sans CJK SC | 16dp |
| 辅助文字 | Noto Sans CJK SC | 12dp |

### 4.2 颜色

| 用途 | 色值 | 说明 |
|------|------|------|
| 主色调 | #607D8B (Blue Gray) | 按钮、标题 |
| 背景色 | #FFFFFF | 页面背景 |
| 辅助文字 | #999999 | 路径、描述 |
| 分隔线 | #E0E0E0 | 列表分隔 |

### 4.3 间距

| 元素 | 间距 |
|------|------|
| 页面内边距 | 10dp |
| 元素间距 | 5-15dp |
| 列表项高度 | 60dp |
| 按钮高度 | 40-50dp |

### 4.4 组件样式

**按钮：**
- 圆角：4dp
- 内边距：8dp
- 背景色：主色调
- 文字色：白色

**列表项：**
- 内边距：8dp
- 分隔线：1dp，辅助色
- 背景色：白色

## 5. 技术约束

### 5.1 Kivy 限制

| 限制 | 解决方案 |
|------|----------|
| 无原生控件 | 自定义绘制所有 UI |
| 中文字体缺失 | 注册 Noto Sans CJK SC |
| 打包体积大 | 使用 PyInstaller 压缩 |

### 5.2 中文字体处理

```python
from kivy.core.text import LabelBase

CHINESE_FONT_PATH = "/usr/share/fonts/opentype/noto/NotoSansCJK-Regular.ttc"
LabelBase.register(name="Chinese", fn_regular=CHINESE_FONT_PATH)

# 使用
Label(text="中文", font_name="Chinese")
```

### 5.3 数据路径

```python
# 默认数据路径
DEFAULT_DATA_PATH = Path(__file__).parent.parent.parent.parent.parent / "meta"

# 使用
submodules = load_submodules(DEFAULT_DATA_PATH)
```

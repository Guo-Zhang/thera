# 沟通语境`Context`

## 定义

沟通语境（Communication Context）是指围绕一次沟通事件的所有背景信息，包括物理环境（地点）、社会关系（身份、角色）、文化背景、情绪氛围，以及之前的对话等，这些元素共同决定了语言的真正含义，使含糊的词语变得清晰，并可能产生超出字面意义的“语境意义”。理解语境对于准确理解和有效沟通至关重要，尤其是在高语境文化（如中国、日本）中，非言语线索和情境信息比直接的语言更重要，而低语境文化（如美国、德国）则更依赖明确的语言信息。

## 建模

不是“说过什么”，而是“我们当前共同相信什么、聚焦什么、忽略什么”。人机信任的界面。
错误定义	正确定义
Context = 对话历史字符串	Context = 显式的协作认知状态
Context 是 AI 的内部黑盒	Context 是人机共享的可协商契约
Context 由系统单方面决定	Context 通过推断 + 修正 + 协商共同维护

当你能清晰看到、修改、确认“我们此刻在想什么”，那种“费力重复语境”的疲惫感就会消失——因为语境不再是负担，而是你们共同握在手中的导航图。

语境（Context）

是当前协作对话中，已被双方共同接受或正在协商的认知状态集合，

它显式描述了：

我们在聚焦什么（Focus）
我们相信什么（Grounded Beliefs）
我们可能走向哪里（Open Possibilities）

而是一个结构化的、带元状态的值对象（Value Object）

这个定义：

可编程（有明确字段和行为）
可协商（带 PENDING/CONFIRMED 状态）
可观察（能直接展示给用户）
可演进（通过不可变更新支持回溯）

但先保证整体语境的显性化与可操作性——这正是你最初直觉的方向。

## 评估

好的语境不是“答案”，而是“高质量的问题容器”。
它既提供理解的坐标系，又保留足够的缝隙，让新的语言、动作、沉默能够自然生长出来。

## 更新模式

优先级从低到高：
1. 自动推断（Auto-inference）：从新消息中提取隐含信息
2. 手动修正（Manual-correction）：用户手动修正已有信息
3. 协商确认（Negotiation-confirmation）：用户与系统协商

## 共识状态

只有 confirmation == CONFIRMED 的语境才用于关键决策（如剧情分支）
PENDING 状态的语境仅用于生成试探性回复，并附带澄清提问

## 语境状态

我们可以将沟通语境的形式化理解为：

Stateₙ = Update(Stateₙ₋₁, Δ)

其中，Δ 是由新对话（utterance）所携带的语境增量（contextual delta），它可能表现为：

增补（addition）：引入新元素（如新角色、新情绪、新隐喻）；
强化（reinforcement）：加深已有维度（如将“温柔”细化为“带着痛感的温柔”）；
修正（revision）：调整先前假设（如原以为关系是单向依赖，现揭示为相互映照）；
否定/覆盖（negation/override）：推翻旧有前提（如“他们彼此信任” → “信任正在崩解”）；
重构（reframing）：改变解释框架（如将“AI的局限”从技术缺陷重释为存在论诗意）。

因此，在建模时，我们不仅要记录“加了什么”，更要标记“改了什么”“删了什么”“如何重新解释旧信息”。这也呼应了语言哲学中的语境敏感性（context-sensitivity）和预设撤销（presupposition cancellation）现象。
好的语境模型，必须允许“此刻推翻昨天”——因为人类沟通的本质，本就充满惊奇、误认与顿悟。

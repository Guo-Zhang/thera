# 工作文档

## 产品需求

推演半自传体小说剧情有较大的推理和记忆工作量，希望使用这个外脑系统帮助我从这些事务脱身专注于创造。

AI辅助的外接大脑帮助深度思考。
我写这个项目的动机是写小说的时候发现推理工作量比较大。
目前困扰我的主要问题是要保证小说逻辑内部的一致性。
首先就要能够把关键信息分门别类地存起来，然后要能够在推理和检查剧情时充分使用这些信息获得一致性。
以现实为原型写半自传体。所以我需要区分现实、虚构，以及建立现实到虚构的映射，并且要维护虚实之间的桥梁。


## 产品特性

- 编辑：可交互式提交知识
- 查看：可查看现有知识
- 批量导入：支持从dev_docs目录批量导入文档作为知识图谱中的Episodes

### 提交知识

用户在对话中输入一个原始想法：

```
我想写一个以滁州为原型的小说，在小说中以滁州的别名亭城为名，实际写作的时候增加很多虚构。
```

调用大模型 API 处理成知识图谱：

```
现实：滁州
虚构：亭城
关系：原型
```

以及定义映射关系：
```
# 映射关系：现实地点 映射_to 虚构地点
映射_to(maps_to)         
```

## 架构设计

### 核心洞察

AI 不缺算力，但缺结构。通过分离知识、记忆与指令，我们可以让 LLM 在可控、一致、可追溯的框架下发挥最大价值。

### 系统分层

用户界面层：提供人与AI交互维护KG的页面。
应用层：调用大模型API处理原始信息并存入图数据库。
基础设施层：大模型API、图数据库等。

### 领域模型

- `methods`：策略库
- `memories`：记忆库
- `prompts`：推理器


## Q&A

1. 哪些是一定需要 AI 辅助的？
在数字资产管理和内容创作领域，尤其是当任务涉及大量数据处理、模式识别或需要基于已有框架进行复杂推理时，AI辅助变得尤为重要。例如，在分类大量的数字资产或将推理应用于已有知识基础中时。

2. 哪个需求优先级高？
如果目标是尽快交付价值，那么与直接产出成果相关的推理工作应该被赋予较高的优先级。这有助于确保项目能够快速迭代并根据反馈调整方向。

3. 有没有具体场景？
您提到了小说推理的例子，这是一种具体的场景。通过给AI提供剧情概要和其他关键信息，可以利用AI生成或推测后续剧情发展，帮助作家突破创作瓶颈或是为读者提供个性化的阅读体验。

4. 这些问题的本质是什么？
本质上，这些问题要求系统既能存储和检索信息（记忆功能），也能基于这些信息做出决策或产生新的想法（推理功能）。AI在这方面有着天然的优势，尤其是在处理非结构化数据和复杂模式识别上。

5. 如何固定这些问题的解决办法？
使用SQL表来组织和查询数据，同时结合提示词（Prompts）指导AI如何处理这些数据。这种方法既利用了数据库的强大查询能力，也发挥了AI在理解和生成自然语言上的优势。

6. 如何开始这些工作？
建立一个提示词文档以标准化输入到AI的信息格式，并创建一个清单文档作为数据库表的替代品，用于记录和管理信息。对于小说剧情推理问题，尝试制作一个简洁的首页文档，概括出向AI提供的关键信息，以便更高效地获得预期结果。

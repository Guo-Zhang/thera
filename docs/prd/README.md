# 产品需求文档

## 用户

苹果，深度知识工作者，每天有一堆脑细胞要费。

## 场景

场景 1：自我探索

需求 1.1：可以持续记录找到模式

功能 1.1.1：记录到知识图谱里
功能 1.1.2：后台分析寻找规律，或者人工辅助寻找规律。
功能 1.1.3：查询分析结果。

场景 2：工作思考

需求 2.1：可以公开给团队的工作思考，转移到团队外脑里。
需求 2.2：可以日夜不停地写代码，或者至少可以审代码。

我想要一个永动机一样的 AI 在不停地写代码做产出。这样可以缓解我不干活就不产出的焦虑感。
理想和现实的差距是，AI 写代码很容易失控，很难离开人干活。
所以我想让 AI 可控一些，哪怕质量不高也可以。

现实一点，比如白天写了很多想法放在哪里，然后晚上交给 AI 自动产出代码做测试。
从工程的角度控制输入输出，而不是控制写代码的过程。

比如 Python 的知识图谱。说不定甚至可以有现成的可以直接用或者修改。
这可以解决当前不会构建知识图谱缺少案例学习的问题。
这个可以直接帮助我写代码。

场景 3：写小说

需求 3.1：可以从前两个场景取材并映射到虚构世界里。

## 需求

### 未分类

需要能导出标准文件
需要能看到案例的推理过程
需要可以建立案例库

## 功能

### 更新知识图谱

根据查询实际需求更新图谱

比如查谁会 Python，根据职业可以推理，但是还是需要模型推理，就可以设法存下来。

自动模式，半自动模式，手动模式。

自动模式为主， AI 操作、人验收。所以难点在于对人透明。
